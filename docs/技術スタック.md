## 3-2. Phase 2-4 で追加導入予定

「凝ったアプリ」化に伴い、Phase 別に以下を導入する。

| Phase | 種別 | ライブラリ / 技術 | 用途 / 採用理由 |
|---|---|---|---|
| **Phase 2** | フロント | **Chart.js** (^4) | 予算 vs 実績の棒・円グラフ (F-BUDGET-03)。Vue 統合は `vue-chartjs` |
| **Phase 2** | バックエンド | (gem 追加なし / 標準 ActiveRecord で完結) | タグ / カテゴリ / フォロー / プランニング / 予算は標準 CRUD |
| **Phase 2** | バックエンド | `acts-as-taggable-on` (代替: 自前 `trip_tags`) | タグ実装の負荷削減。**注意**: DSL が独特で学習負荷高め (`acts_as_taggable_on :tags` のメタプログラミング理解必要)。**推奨は自前 `trip_tags`** (ER 図 §5 設計と整合性が高く、Rails 標準で透過的) |
| **Phase 3** | フロント | **Leaflet** (^1.9) + OpenStreetMap タイル | 地図ピン表示 / ルート描画 (F-MAP-01〜03)。MapLibre GL JS は WebGL 必須で代替案 |
| **Phase 3** | フロント | **Nominatim API** (geocoding) | 緯度経度の住所検索 (F-MAP-02)。OSS / レート制限あり |
| **Phase 3** | バックエンド | **`prawn`** + `prawn-table` (推奨) | PDF エクスポート (F-EXPORT-01/02)。Ruby ネイティブで wkhtmltopdf 不要。日本語フォント埋め込み要 (IPA 等)。代替の `wicked_pdf` は HTML 流用できるが wkhtmltopdf がメンテ停止のため非推奨 |
| **Phase 3** | バックエンド | **AWS SDK for Ruby (S3)** | 画像保存先を Disk → S3 へ (F-S3-01) |
| **Phase 3** | バックエンド | **`rack-attack`** | ログイン rate limit (E-M1 残課題) |
| **Phase 3** | バックエンド | **独自 JwtDenylist テーブル + 既存 `JsonWebToken` サービス拡張** | JWT revoke (E-M2 残課題)。**`devise-jwt` は既存自前 `JsonWebToken` (`app/services/json_web_token.rb`) と二重化リスクがあり不採用**。`JsonWebToken.decode` の前段で denylist 確認するだけで実装可能 |
| **Phase 4** | バックエンド | **Action Cable** (Rails 同梱) + `solid_cable` (PR #6 で導入済) | リアルタイム反映 + DM (F-RT-01 / F-DM-*) |
| **Phase 4** | フロント | **Workbox** (PWA) | Service Worker でオフライン閲覧 (F-PWA-01) |
| **Phase 4** | フロント / バック | **`rails-i18n`** + Vue I18n | 多言語 (F-I18N-01) |
| **Phase 4** | バックエンド | **`web-push`** gem | Web Push 通知 (F-NOTIF-03) |
| **Phase 2** | フロント | **`vuedraggable` (Sortable.js 系)** | DayEntry / 荷物 / planned_spots の D&D 並び替え (F-DAY-DRAG / F-PACK-01) |
| **Phase 2** | フロント | **`@vueuse/core`** (useLocalStorage / useIntersectionObserver) | フォーム自動保存 / 画像遅延読み込み / ダークモード設定 (F-UX-AUTOSAVE / F-UX-LAZY / F-UI-DARK) |
| **Phase 2** | フロント | **Tailwind `dark:` バリアント** | ダークモード (F-UI-DARK)。`darkMode: 'class'` 設定 |
| **Phase 2** | バックエンド | (gem 追加なし) | 下書き (trips.status enum) / お気に入り / メモ / 荷物 / チケット / レビュー / 検索履歴 は標準 ActiveRecord で完結 |
| **Phase 3** | バックエンド | **OSRM Public API** (オープンソース ルーティング) | 2 地点間のルート取得 (F-MAP-06)。**注意**: Public Demo Server は商用利用不可 / レート制限厳しい / SLA なし。本格運用は (a) 自前 OSRM を Docker でホスティング (OSM データ取り込み要) または (b) Mapbox Directions / Google Maps Directions の商用 API (有料) を採用。学習用 (講師提出) は Demo Server で許容 |
| **Phase 3** | フロント | **`svgmap` + 自前都道府県 SVG** (or `topojson-jp`) | 日本地図で都道府県制覇率ハイライト (F-STATS-02 / F-MAP-04) |
| **Phase 3** | フロント | **`PhotoSwipe`** または **`vue-easy-lightbox`** | アルバム画像 Lightbox 拡大 (F-ALBUM-01) |
| **Phase 4** | バックエンド | **Rails `ActiveRecord::Locking::Optimistic`** (gem 追加なし) | 楽観ロック (F-GROUP-02)。`trips` に `lock_version` を追加するだけ |

## 4. 採用しない技術と理由

| 候補技術 | 不採用の理由 |
|---|---|
| TypeScript | 学習段階 (習う→慣れる) で型定義に時間を取られると本筋が止まる。Phase4 で導入余地あり |
| GraphQL | REST で十分な規模。Rails の標準と相性が良い REST を選択 |
| Redis | キャッシュ / セッション要件が Phase 1〜2 では不要 |
| Sidekiq (非同期 Job) | 同期処理で十分。Phase3 で通知の非同期化を検討 |
| Docker (Rails / Nuxt) | 開発体験 (rbenv / npm の即時反映) を優先。DB のみ Docker |
