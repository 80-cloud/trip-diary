### 1-3. 環境変数 (`.env`)

`.env.example` をコピーして `.env` を作成し、各種シークレットを設定する。**`.env` は絶対にコミットしない**。

```env
MYSQL_ROOT_PASSWORD=...
MYSQL_DATABASE=trip_diary_dev
MYSQL_USER=trip
MYSQL_PASSWORD=...
MYSQL_PORT=3316
RAILS_PORT=3010
SECRET_KEY_BASE=...        # bundle exec rails secret
JWT_SECRET=...             # openssl rand -hex 64
NUXT_PORT=3011
NUXT_PUBLIC_API_BASE=http://localhost:3010/api/v1
```

### 2-2. AWS リソース (Terraform で IaC 化 / `infra/*.tf`)

| リソース | スペック | 月額目安 | 備考 |
|---|---|---|---|
| ECS Fargate (Rails task) | 256 CPU / 512-1024 MB / 1 task | ~$9 | Backend (Rails 8.1 / Docker / port 3000) |
| ECS Cluster / Service | - | 無料 | Fargate launch type |
| ECR (private) | trip-diary-backend / image tag = git SHA | 無料枠内 (500MB) | latest tag 禁止 (mutable tag prod 誤投入防止) |
| ALB | listener 443 (ACM) + tg/rails | ~$18 | health check `/api/v1/health` |
| RDS | db.t3.micro / MySQL 8 / 20GB / Multi-AZ なし | 無料 (12ヶ月) | 自動スナップショット 7 日保持 |
| S3 uploads bucket | OAC 経由 CloudFront 配信 | ~$0.5 | Active Storage (`config.active_storage.service = :amazon`) |
| S3 frontend bucket | CloudFront origin | ~$0.5 | Nuxt SPA (`npm run generate`) の出力を sync |
| CloudFront | PriceClass_200 (日本含むアジア) | ~$2 | frontend SPA + /images/* (uploads OAC) |
| ACM 証明書 | 1 ドメイン (Route53 DNS 検証) | 無料 | ALB と CloudFront 用 (us-east-1 リージョン必要) |
| Route53 | A レコード × 2 (root + api) | $0.5 | ドメインは既存取得済 |
| SSM Parameter Store | SecureString (KMS alias/aws/ssm) | 無料枠内 | RAILS_MASTER_KEY / DATABASE_URL / JWT_SECRET / SECRET_KEY_BASE / CORS_ORIGINS / S3_BUCKET / S3_REGION |
| IAM | execution_role + task_role | 無料 | task_role に s3:* (uploads bucket スコープ) + ssm:GetParameters + kms:Decrypt |
| CloudWatch Logs | log group `/ecs/trip-diary-backend` | 無料枠内 (5GB/月) | Rails STDOUT 出力をそのまま |
| VPC / Subnet / SG / IGW / NAT | a + c の 2 AZ | 無料 | NAT は private subnet なし設計のため省略可 |
| AWS Budgets (任意) | 月次 $30 上限通知 | 無料 | Phase B で導入 |

**月額合計概算**: ~$30 (無料枠 12ヶ月内は ~$20)

### 2-3. デプロイフロー (sns-board の scripts/ 流用)

```
[backend (Rails)]                          [frontend (Nuxt SPA)]
git commit                                 git commit
    │                                          │
    ▼ scripts/deploy-backend.sh                ▼ scripts/deploy-frontend.sh
docker buildx --platform linux/amd64       npm run generate
    │                                          │
    ▼ aws ecr push (tag=git SHA)               ▼ aws s3 sync .output/public s3://...
terraform apply -var backend_image_tag=...     │
    │ (Task Def 更新)                          ▼ aws cloudfront create-invalidation /index.html
    ▼
aws ecs update-service --force-new-deployment
    │
    ▼ wait services-stable (rolling update)
```

> **重要**: image tag は必ず git commit SHA (`latest` 禁止)。mutable tag を本番に誤投入する事故防止 ([docs/セキュリティ自己監査.md](./セキュリティ自己監査.md) E-H3 と同型のリスク)。

### 2-4. セキュリティ・運用

| 項目 | 方針 |
|---|---|
| 通信 | 全リスナー HTTPS only (ACM / ALB は HTTP → HTTPS リダイレクト) |
| DB アクセス | RDS SG は ECS task SG からのみ 3306 許可 |
| 機密情報 | SSM Parameter Store (SecureString / KMS 暗号化) / 静的 AWS キー不要 (IAM Role 経由) |
| 画像 IAM | task_role に `s3:PutObject` / `GetObject` / `DeleteObject` を uploads bucket スコープで付与 |
| バックアップ | RDS 自動スナップショット (7 日保持) / S3 versioning は Phase B 検討 |
| ログ | Rails STDOUT → CloudWatch Logs (awslogs ドライバ) |
| 監視 | CloudWatch アラーム (CPU > 80% / 5xx 急増 / RDS IOPS) は **Phase B** で導入 |
| デプロイ | image tag = git SHA / 鍵類は SSM 経由 / `terraform plan` は人間目視必須 (CLAUDE.md §6) |

## 3. コスト管理

- 想定: 学習用 + 講師レビュー → 無料枠 12ヶ月内で完結 (~$20/月)
- 検証完了後は `terraform destroy` (`force_destroy_resources = true` を tfvars で指定して S3/ECR 中身ごと削除可能化)
- AWS Budgets で月 $30 上限通知 (Phase B / sns-board の `budgets.tf` を port)

## 4. 障害対応 (Phase3 で詳細化予定)

- 5xx 急増時: Nginx / Puma / Rails ログを CloudWatch で確認
- DB 接続不可: RDS コンソール / SG 設定確認
- ディスク容量逼迫: 画像 S3 化 / ログローテーション
- 復旧手順: `OPERATIONS.md` を Phase3 で整備 (recipe-board の体系を流用)
