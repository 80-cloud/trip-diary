## §1 目的とスコープ

- 「習う→慣れる→マスター」 ([学習ロードマップ.md](./学習ロードマップ.md) §1) の **慣れ** 段階で、**テストが品質の防波堤** であることを言語化する
- Phase 1 は基盤 + サンプルのみ。Phase 2 で本番品質に引き上げ、Phase 3 では CI ゲート化
- **本書スコープ**: backend (Rails Minitest) / frontend (Vitest) / 手動ブラックボックス
- **範囲外**: E2E (Playwright/Cypress 不採用) / 性能 / 負荷 (Phase 4 検討)

```bash
cd backend

# 出力例: 4 runs, X assertions, 0 failures
```

- **fixtures**: BCrypt cost: 4 で高速化 ([test/fixtures/users.yml](../backend/test/fixtures/users.yml) のコメント参照)

## §8 手動ブラックボックスチェックリスト (各 PR マージ前に実施)

Playwright を採用していないため、**本リストが UI 回帰の最終防衛線**。Chrome で実施 (必要に応じて Safari)。

### 認証フロー
- [ ] `/signup` で新規登録 → トップに遷移
- [ ] `/login` (正常) → 200 / トップに遷移 / ヘッダーに display_name 表示
- [ ] `/login` (誤 email) → 401 / 文言「メールアドレスまたはパスワードが間違っています」
- [ ] `/login` (誤 password) → 401 / **同じ文言** (email と区別されない / E-H1 確認)
- [ ] 未ログインで `/trips/new` → `/login` リダイレクト
- [ ] ログアウト → Cookie 削除 / `/me` が `{user: null}`

### Trip CRUD
- [ ] 一覧 `/` が public trip を新しい順に表示
- [ ] `/trips/new` で title/destination/期間/本文/出来事 (ネスト) 入力 → 作成 → 詳細遷移
- [ ] `/trips/:id/edit` で自分の trip 編集 → 反映
- [ ] **他人の trip の `/edit` を URL 直打ち → 403** (認可確認)
- [ ] 自分の trip 削除 → 一覧から消える

### コメント / いいね
- [ ] 詳細でコメント投稿 → リスト末尾に表示
- [ ] 他人のコメントに削除ボタンが**出ない**
- [ ] いいねボタンクリック → 数値+1 / 再クリック → -1 (2 回目以降の POST は 200 で冪等)

### 画像
- [ ] 6 枚アップロード → 「最大 5 枚まで」エラー
- [ ] 6MB 画像 → 「5MB 以下」エラー

### XSS / 入力
- [ ] title に `<script>alert(1)</script>` → エスケープ表示 (alert 出ない)

### セキュリティ (DevTools)
- [ ] Network パネルで Request Headers に **Authorization が露出していない** (Cookie 認証のため)
- [ ] Application パネルで Cookie の `HttpOnly` フラグが ON
