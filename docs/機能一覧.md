## 機能詳細 (Phase 1 のみ抜粋)

### F-AUTH-01: サインアップ

- 入力: email (必須・形式チェック・unique) / password (必須・6 文字以上) / display_name (必須・1〜30 文字)
- 動作:
  1. Rails 側で `User.create!` (パスワードは BCrypt でハッシュ)
  2. 成功時に JWT を発行し HttpOnly Cookie にセット
  3. フロントは `/` (タイムライン) へ遷移
- エラー:
  - email 重複: `422 { error: "メールアドレスは既に使用されています" }`
  - バリデーション失敗: `422 { errors: { ... } }`

### F-AUTH-02: ログイン

- 入力: email / password
- 動作:
  1. `User.find_by(email:)` → `authenticate(password)`
  2. 成功時に JWT を HttpOnly Cookie へセット (有効期限 1 日 — Phase2 で access + refresh の 2 段方式へ拡張)
  3. レスポンス: `200 { user: { id, email, display_name } }`
- エラー: `401 { error: "メールアドレスまたはパスワードが間違っています" }`

### F-TRIP-03: 旅行記録 作成

- 入力 (1 画面で全部):
  - title (必須・1〜80 文字)
  - destination (必須・1〜80 文字 — 例: 「京都」「ハワイ・オアフ島」)
  - started_on / ended_on (必須・started ≤ ended)
  - body (任意・5000 文字以内)
  - day_entries (任意・複数):
    - happened_on (任意 — 期間内推奨)
    - title (必須・1〜80 文字)
    - body (任意・2000 文字以内)
  - images (任意・最大 5 枚・各 5MB 以下)
- 動作:
  - `Trip.create!` + `day_entries_attributes` (accepts_nested_attributes_for)
  - 画像は `trip.images.attach(...)` (ActiveStorage)
- 認可: ログイン必須。自分の `user_id` で固定 (リクエスト body から user_id は受け取らない)

### F-CMT-01 / F-LIKE-01: コメント・いいね

- 認可: ログイン必須
- いいねは `(trip_id, user_id)` の複合 unique 制約で 1 回まで
- 投稿時に `trip.comments_count` / `trip.likes_count` を counter_cache で更新

## 機能間の依存関係

```
F-AUTH-* ──┬─→ F-TRIP-03/04/05 (投稿者の特定)
           ├─→ F-CMT-01/02 (コメント投稿者の特定)
           └─→ F-LIKE-01/02 (いいねユーザーの特定)

F-TRIP-* ──┬─→ F-DAY-* (Trip 配下のネスト)
           ├─→ F-CMT-* (Trip 配下のコメント)
           └─→ F-LIKE-* (Trip 配下のいいね)
```
