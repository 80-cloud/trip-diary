## 5. バリデーション一覧

| 画面 | 項目 | ルール | エラーメッセージ例 |
|---|---|---|---|
| S-01/02 | email | 必須 / 形式 | 「メールアドレスを正しく入力してください」 |
| S-01/02 | password | 必須 / 6 文字以上 | 「パスワードは 6 文字以上で入力してください」 |
| S-02 | display_name | 必須 / 1〜30 文字 | 「表示名は 30 文字以内で入力してください」 |
| S-05 | title | 必須 / 1〜80 文字 | 「タイトルは 80 文字以内で入力してください」 |
| S-05 | destination | 必須 / 1〜80 文字 | 「行き先は 80 文字以内で入力してください」 |
| S-05 | started_on / ended_on | 必須 / started ≤ ended | 「終了日は開始日以降を指定してください」 |
| S-05 | body | 任意 / 5000 文字以内 | 「本文は 5000 文字以内で入力してください」 |
| S-05 | day_entries.title | 出来事を追加した場合は必須 / 1〜80 文字 | 「出来事のタイトルを入力してください」 |
| S-05 | images | 任意 / 最大 5 枚 / 各 5MB | 「画像は最大 5 枚までです」 |
| S-04 | comment.body | 必須 / 1〜140 文字 | 「コメントは 140 文字以内で入力してください」 |

## 6. エラー / 空状態 デザインルール

| 状態 | 表現 |
|---|---|
| 一覧 0 件 | 「まだ旅行記録がありません。最初の記録を残しましょう」 + 新規作成ボタン |
| 404 (詳細が見つからない) | 「この旅行記録は見つかりませんでした」 + 一覧へ戻るリンク |
| 401 (未ログインで保護画面) | ログイン画面へリダイレクト |
| 403 (他人の編集画面) | 詳細画面へリダイレクト + トースト「自分の旅行記録のみ編集できます」 |
| 500 (サーバーエラー) | 「サーバーで問題が発生しました。時間を置いて再度お試しください」 |

## 7. レスポンシブ方針 (Phase1)

- ブレークポイントは Tailwind デフォルト (sm:640 / md:768 / lg:1024 / xl:1280)
- カード一覧: md 以上で 2 カラム / lg 以上で 3 カラム / それ未満で 1 カラム
- フォーム: md 以上で 中央寄せ最大 720px / それ未満で 100%
- スマホ専用最適化は Phase3
