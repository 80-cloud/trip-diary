## Phase 3 スコープ確定経緯 (v0.4 / 2026-05-17)

### 経緯

v0.3 までは Phase 3 に「地図 / 通知 / PDF / 統計 / アルバム / PWA / S3 / AWS デプロイ」を含む 20+ 機能を集約していた。しかし以下の事情で **講師提出スコープを明確化** する必要が生じた:

1. **テスト先行ルール (CLAUDE.md §「テスト先行原則」)** — 講師提出前に E2E + perf 基盤を整備することが必須となり、Phase 3 にこれらを追加
2. **本番投入のリスクと工数** — 機能数を絞らないと再デプロイ + 講師提出までの期間に収まらない
3. **要件定義書 §1-5 提出物** — 必須は「GitHub / README / docs / CI / PR 履歴 / 本番 URL」であり、追加機能は加点要素

## 不採用機能 (参考資料 / 将来再検討候補)

学習価値と動作担保のトレードオフで本書のスコープ外と判断した候補。将来要件が変わったら本セクションから引き上げる。

| 候補 | 不採用理由 | 再検討トリガー |
|---|---|---|
| **混雑表示 / ライブ混雑** | リアルタイム混雑データの取得元なし (Google Places は有料 API / OSS で広域な混雑データなし) → 動作担保困難 | 公的混雑 API の登場 / 有料 API 採用許容 |
| **店舗自動取得 / コンビニ表示 / 道の駅表示** | 外部 API 依存 (Google Places 有料 / Overpass API レート制限厳しい) | 商用 API 採用 / OSM Nominatim で代替案策定 |
| **オフライン地図** | タイル全部キャッシュは大容量 + タイル提供元のライセンス問題 | MBTiles + 自前タイルサーバ (Phase 4 以降の発展枠で検討) |
| **災害情報表示** | 旅行記録の主旨外 / JMA API 取得自体は可能 | 旅行アプリではなく防災アプリ寄りに方針転換した場合 |
| **クラウド同期** | 既に DB 保存 = 実質クラウド同期。意味不明確 | (該当なし) |
| **自動カテゴリ分類** | ML 必要 / 学習範囲外。Phase 4 でも単純なキーワードマッチング止まり | LLM API 採用許容 (OpenAI / Claude) |
| **到着通知 / 移動軌跡記録** | Web ではなくモバイルネイティブ App 領域 (GPS 連続監視 + Geofence) | React Native / Flutter で別アプリ化 |
| **バッテリー節約モード** | Web の Battery Status API は廃止傾向 / 効果も限定的 | (該当なし) |
| **屋外 iPhone でも使える** | PWA + レスポンシブ設計 (F-PWA-01) で内包済 | (内包済のため再検討不要) |

## 機能詳細 (Phase 1 のみ抜粋)

### F-AUTH-01: サインアップ

- 入力: email (必須・形式チェック・unique) / password (必須・6 文字以上) / display_name (必須・1〜30 文字)
- 動作:
  1. Rails 側で `User.create!` (パスワードは BCrypt でハッシュ)
  2. 成功時に JWT を発行し HttpOnly Cookie にセット
  3. フロントは `/` (タイムライン) へ遷移
- エラー:
  - email 重複: `422 { error: "メールアドレスは既に使用されています" }`
  - バリデーション失敗: `422 { errors: { ... } }`

### F-AUTH-02: ログイン

- 入力: email / password
- 動作:
  1. `User.find_by(email:)` → `authenticate(password)`
  2. 成功時に JWT を HttpOnly Cookie へセット (有効期限 1 日 — Phase2 で access + refresh の 2 段方式へ拡張)
  3. レスポンス: `200 { user: { id, email, display_name } }`
- エラー: `401 { error: "メールアドレスまたはパスワードが間違っています" }`

### F-TRIP-03: 旅行記録 作成

- 入力 (1 画面で全部):
  - title (必須・1〜80 文字)
  - destination (必須・1〜80 文字 — 例: 「京都」「ハワイ・オアフ島」)
  - started_on / ended_on (必須・started ≤ ended)
  - body (任意・5000 文字以内)
  - day_entries (任意・複数):
    - happened_on (任意 — 期間内推奨)
    - title (必須・1〜80 文字)
    - body (任意・2000 文字以内)
  - images (任意・最大 5 枚・各 5MB 以下)
- 動作:
  - `Trip.create!` + `day_entries_attributes` (accepts_nested_attributes_for)
  - 画像は `trip.images.attach(...)` (ActiveStorage)
- 認可: ログイン必須。自分の `user_id` で固定 (リクエスト body から user_id は受け取らない)

### F-CMT-01 / F-LIKE-01: コメント・いいね

- 認可: ログイン必須
- いいねは `(trip_id, user_id)` の複合 unique 制約で 1 回まで
- 投稿時に `trip.comments_count` / `trip.likes_count` を counter_cache で更新
